# NBA 比賽預測

此專案應該嚴格使用 `pydantic` 來定義數據型態，確保數據的完整性與一致性
數據型態應該放在 `./src/types/` 目錄底下分門別類以達到良好的組織結構
代碼中的所有註解應該為英文, 但最後總結給 User 時必須為繁體中文

- [] 透過 `nba_api` 蒐集, 整理, 和存儲數據以下數據

    - [ ] 基礎數據 (得分、籃板、助攻、命中率、上場時間)
    - [ ] 高階數據 (USG%、Net Rating、TS%)
    - [ ] 球員狀態 (出勤率、生涯傷病頻率、輪休紀錄)
    - [ ] 賽程負荷 (休息天數、是否為 B2B、賽季進度)
    - [ ] 球隊環境 (隊友缺陣時的數據變化)
    - [ ] 設計數據庫結構
    - [ ] 數據清洗與預處理
    - [ ] 特徵工程與選擇

    - [ ] 選擇適合的演算法 (如迴歸、決策樹、隨機森林等)

    - [ ] 評估模型效能 (使用交叉驗證、混淆矩陣等方法)
